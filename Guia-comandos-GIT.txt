## ****Criando seu primeiro repositório****

### ****Configurações iniciais do Git****

**Baixando e instalando o Git**

<aside>
💡 https://git-scm.com/downloads

</aside>

**Configura nome de usuário**

<aside>
💡 git config --global user.name nome-sobrenome

</aside>

**Configura email de usuário**

<aside>
💡 git config --global user.email email@email.com.br

</aside>

## ****Inicializando um repositório****

**Inicializa o versionamento no respectivo diretório**

<aside>
💡 `git init`

</aside>

## ****Comandos básicos para sobreviver****

**Verifica o status do repositório**

<aside>
💡 `git status`

</aside>

**Adiciona todos os arquivos para serem commitados**

<aside>
💡 `git add .`

</aside>

**Commitando arquivos**

<aside>
💡 `git commit -m "inserir um comentário significativo"`

</aside>

**Visualizando relatório de commits**

<aside>
💡 git log // todos os commits
git log --oneline // exibe log com hash e título do commit

</aside>

**Adicionando um repositório remoto**

<aside>
💡 `git remote add origin https://github.com/User/Repository.git`

</aside>

**Enviando as modificações para o repositório remoto**

<aside>
💡 `git push origin <branch>`

</aside>

**Puxando alterações do repositório remoto**

<aside>
💡 `git pull origin <branch>`

</aside>

## ****Trabalhando com branchs****

**Criando e locomovendo-se para uma nova branch**

<aside>
💡 `git checkout -b nome-branch`

</aside>

**Aplicando merge em branchs**

<aside>
💡 `git merge nome-branch // precisa estar na branch de destino`

</aside>

**Visualizando todas as branches existentes no repositório**

<aside>
💡 `git branch`

</aside>

**Deletando uma branch local**

<aside>
💡 `git branch -D nome-branch`

</aside>

**Deletando uma branch remoto**

<aside>
💡 `git push origin :nome-branch`

</aside>

### ****Repositórios****

**Repositórios locais**

Um repositório nada mais é do que uma pasta onde estão todos os arquivos do nosso projeto, incluindo nosso versionamento. Neste caso, o nosso repositório vai ser armazenado de forma local em nossa máquina, impossibilitando que outras pessoas tenham acesso.

**Repositórios remotos**

Um repositório remoto é criado em um local de hospedagem de repositórios Git, neste caso o GitHub, onde podemos armazenar uma cópia do nosso projeto na nuvem.
